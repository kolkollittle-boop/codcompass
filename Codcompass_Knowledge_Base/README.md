# Codcompass Knowledge Base · 2.0

Codcompass 2.0 reframes the library from **numbered curriculum lanes** into **decision-oriented domains**: places senior engineers go when trade-offs matter more than tool tutorials.

## Philosophy

- **Architecture over anecdotes.** Knowledge is grouped by the kinds of systems and bets you make—not by release order or chapter numbers.
- **Explicit trade-offs.** Patterns are useful only when failure modes, ownership boundaries, and migration costs are visible.
- **Compounding leverage.** Productivity tips sit beside methodology because lasting speed comes from systems, not hacks.
- **Cross-cutting honesty.** Security, cost, and observability are not “later phases”; they shape every roadmap from day zero.

## How to Navigate

| Domain | Intent |
|--------|--------|
| **AI Engineering** | Shipping agents, retrieval systems, local inference economics, and durable AI products—not demos. |
| **System Evolution** | Modernizing estates, scaling backends responsibly, and turning infrastructure into repeatable engineering. |
| **Platform & Product Engineering** | Solo or small teams operating like platforms: delivery loops, distribution, and automation. |
| **Developer Mastery** | Workflow design, disciplined learning, and architectural cognition—the human multiplier. |
| **Cross-Cutting Concerns** | Controls and realities that intersect every pillar: trust boundaries, unit economics, signals. |
| **Insights & Methodology** | Canon, synthesis, and research-grade notes that refine judgment. |
| **Archive** | Superseded yet informative material—kept for lineage, not active recommendation. |
| **References** | External anchors, standards, and citation-grade sources that stabilize claims. |

Each topical folder contains an `index.md` with **mental models**, **decision frameworks**, and a **maturity path** so you can orient before drilling into articles.

Start with `MAP.md` for a structural diagram, then open any `index.md` that matches the decision you are facing today.
