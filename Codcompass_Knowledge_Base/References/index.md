# References

## Purpose

External **standards, papers, manuals, and authoritative specifications** that stabilize claims elsewhere in the knowledge base. Prefer stable URLs, DOIs, version pins, and archival snapshots where link rot is likely.

## Core Theory & Mental Models

- **References bound debate.** When disagreement stalls progress, elevate to primary documents—not opinions.
- **Version pinning is integrity.** “Latest docs” drift; record the edition your argument depended upon.

## Key Patterns & Decision Frameworks

- **Citation granularity:** Link to sections and anchors when possible; future readers should land on evidence, not homepages.
- **Trust tiers:** Label references as normative (standards), descriptive (vendor docs), or exploratory (blogs)—each plays a different role.

## Learning Path & Maturity Model

1. **Foundational:** Central bibliography format shared across notes (title, version, retrieval date, archive link).
2. **Intermediate:** Topic-specific reference shelves synced with active pillars (AI, systems, compliance).
3. **Advanced:** Automated link health monitoring with ownership for remediation.
