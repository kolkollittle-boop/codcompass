# WebGPU: The Secret Weapon for Browser Video Processing

## Why the CPU-first approach breaks down

A single 4K frame has more than 8 million pixels. At 60fps, that's nearly 500 million pixels per second — before blur, color correction, or motion analysis enter the picture.

Video processing on the CPU works for demos. It falls apart at scale.

The CPU excels at control flow, application logic, scheduling, and parsing. It is not where you want to run millions of identical pixel operations every frame from JavaScript. A GPU is built for exactly that: thousands of small operations running in parallel over images, buffers, and textures.

## The pipeline that works: WebCodecs + WebGPU

Here's the architecture that actually scales:

1. **WebCodecs** decodes video, producing `VideoFrame` objects
2. **WebGPU** imports them via `GPUDevice.importExternalTexture({ source: videoFrame })`
3. Frame becomes an internal `rgba8unorm` texture — stable input for all processing
4. Effects chain through WGSL shaders, staying entirely on GPU
5. Analysis passes (histogram, waveform, optical flow) compute on GPU, returning only compact statistics to CPU

The key insight: **no `getImageData()`, no `readPixels()`, no full-frame CPU readback.** Reading from GPU forces CPU-GPU synchronization — the exact opposite of what you want for real-time preview.

## Effects as fragment shaders

Every effect — hue shift, brightness, contrast, pixelate, kaleidoscope, RGB split, box blur — is a WGSL fragment shader. Sample the input texture, compute new pixel values, write to render target. The pattern is universal.

For effect chains, ping-pong textures swap input/output after each pass. One effect or five, the architecture stays the same.

## Beyond filters: GPU analysis

The project doesn't stop at visual effects. It computes histogram, waveform, vectorscope, and optical flow — all on GPU. The optical flow pipeline runs grayscale conversion, pyramid downsampling, Lucas-Kanade flow — all compute shaders. CPU only receives compact statistics.

## The result

A local-first video processing lab that runs entirely in the browser. No server, no native dependencies. Real-time effects, live preview, and professional-grade analysis tools.
