## 📦 技术栈详情

### 前端 (Frontend)

| 技术 | 选型 | 理由 |
|------|------|------|
| **框架** | Next.js 14 (App Router) | SSR + 静态生成，SEO 友好 |
| **语言** | TypeScript | 类型安全，开发体验好 |
| **样式** | Tailwind CSS | 快速开发，一致性强 |
| **组件** | shadcn/ui | 高质量可定制组件 |
| **图标** | Lucide React | 简洁一致 |
| **状态** | Zustand | 轻量简单 |
| **表单** | React Hook Form + Zod | 类型安全表单验证 |

### 后端 (Backend)

| 技术 | 选型 | 理由 |
|------|------|------|
| **后端** | Next.js API Routes | 前后端同构，部署简单 |
| **认证** | NextAuth.js (Auth.js) | 支持多 Provider |
| **数据库 ORM** | Prisma | 类型安全，开发效率高 |

### 数据库 (Database)

| 技术 | 选型 | 理由 |
|------|------|------|
| **主数据库** | PostgreSQL (Supabase) | 免费额度高，功能完整，自带全文搜索 (tsvector) |
| **缓存/搜索** | PostgreSQL 内置 | MVP 阶段无需 Redis/Meilisearch，使用 Vercel Edge Cache + PG tsvector |

### 内容生产 (Content Pipeline)

| 技术 | 选型 | 理由 |
|------|------|------|
| **爬虫框架** | OpenClaw | 已有部署，支持分布式 |
| **AI 处理** | Hermes Agent | 已有部署，支持多模型 |
| **AI 模型** | Claude Sonnet 4 | 代码理解能力强 |
| **模型备选** | Qwen2.5-Coder | 本地部署，成本低 |

### 支付 (Payment)

| 技术 | 选型 | 理由 |
|------|------|------|
| **支付平台** | Lemon Squeezy | 个人可用，处理税务 |
| **备选** | Paddle | MoR，功能类似 |

### 部署 (Deployment)

| 技术 | 选型 | 理由 |
|------|------|------|
| **前端托管** | Vercel | Next.js 官方，免费额度高 |
| **图片存储** | Vercel Blob | 与 Vercel 集成好 |
| **域名 DNS** | Cloudflare | 免费 CDN，DNS 管理 |

### 监控 (Monitoring)

| 技术 | 选型 | 理由 |
|------|------|------|
| **错误追踪** | Sentry | 免费额度够用 |
| **分析** | Vercel Analytics | 内置，隐私友好 |
| **SEO** | Vercel OG | 动态生成社交图片 |

## 🗄️ 数据库设计 (初稿)

### 核心表

```prisma
// users - 用户表
model User {
  id            String    @id @default(cuid())
  email         String    @unique
  name          String?
  image         String?
  emailVerified DateTime?
  
  subscriptions Subscription[]
  bookmarks     Bookmark[]
  createdAt     DateTime  @default(now())
  updatedAt     DateTime  @updatedAt
}

// subscriptions - 订阅表
model Subscription {
  id              String   @id @default(cuid())
  userId          String
  user            User     @relation(fields: [userId], references: [id])
  
  lemonSqueezyId  String   @unique
  orderId         Int
  variantId       Int
  status          String   // active, paused, cancelled
  
  planType        String   // builder, pro, enterprise
  price           Int
  currency        String
  
  renewsAt        DateTime?
  endsAt          DateTime?
  
  createdAt       DateTime @default(now())
  updatedAt       DateTime @updatedAt
}

// articles - 文章表
model Article {
  id            String   @id @default(cuid())
  slug          String   @unique
  title         String
  description   String?
  content       String   @db.Text
  
  difficulty    String   // beginner, intermediate, advanced, expert
  category      String
  tags          String[]
  
  viewCount     Int      @default(0)
  isPublished   Boolean  @default(false)
  isPremium     Boolean  @default(true)
  
  sourceUrl     String?  // 原始来源
  sourceSite    String?  // 来源网站
  
  authorId      String?
  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}

// bookmarks - 收藏表
model Bookmark {
  id        String  @id @default(cuid())
  userId    String
  articleId String
  
  user      User    @relation(fields: [userId], references: [id])
  article   Article @relation(fields: [articleId], references: [id])
  
  createdAt DateTime @default(now())
  
  @@unique([userId, articleId])
}
```

## 🔌 关键集成

### 1. Lemon Squeezy Webhook

```typescript
// app/api/webhooks/lemonsqueezy/route.ts
export async function POST(req: Request) {
  const payload = await req.json();
  const signature = req.headers.get('x-signature');
  
  // 验证签名
  if (!verifySignature(payload, signature)) {
    return new Response('Invalid signature', { status: 401 });
  }
  
  // 处理事件
  switch (payload.meta.event_name) {
    case 'subscription_created':
      await handleSubscriptionCreated(payload);
      break;
    case 'subscription_updated':
      await handleSubscriptionUpdated(payload);
      break;
    case 'subscription_cancelled':
      await handleSubscriptionCancelled(payload);
      break;
  }
  
  return new Response('OK');
}
```

### 2. NextAuth 认证

```typescript
// lib/auth.ts
import NextAuth from 'next-auth';
import Google from 'next-auth/providers/google';
import Email from 'next-auth/providers/email';

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [
    Google({
      clientId: process.env.GOOGLE_CLIENT_ID,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET,
    }),
    Email({
      server: process.env.EMAIL_SERVER,
      from: process.env.EMAIL_FROM,
    }),
  ],
  callbacks: {
    async session({ session, user }) {
      session.user.id = user.id;
      return session;
    },
  },
});
```

### 3. 内容权限控制

```typescript
// middleware.ts
import { auth } from '@/lib/auth';
import { getSubscription } from '@/lib/db';

export default auth(async (req) => {
  const { nextUrl } = req;
  const session = req.auth;
  
  // 付费内容检查
  if (nextUrl.pathname.startsWith('/kb/premium')) {
    if (!session) {
      return NextResponse.redirect(new URL('/login', nextUrl));
    }
    
    const subscription = await getSubscription(session.user.id);
    if (!subscription || subscription.status !== 'active') {
      return NextResponse.redirect(new URL('/pricing', nextUrl));
    }
  }
});

export const config = {
  matcher: ['/kb/:path*', '/dashboard/:path*'],
};
```

## 📋 开发环境搭建

### 前置要求

```bash
# Node.js
node >= 18.17

# 包管理器
pnpm >= 8.0

# 数据库
# Supabase (云端) 或本地 PostgreSQL
```

### 快速开始

```bash
# 1. 克隆项目
git clone <repo-url>
cd cyberpunk-web

# 2. 安装依赖
pnpm install

# 3. 复制环境变量
cp .env.example .env.local

# 4. 填充环境变量
# 编辑 .env.local 填入 API keys

# 5. 设置数据库
pnpm db:push

# 6. 启动开发服务器
pnpm dev
```
