# Developer Cognitive Systems

## Core Theory & Mental Models

- **Attention as scarce infrastructure**: onboarding, navigation, and noise reduction are architectural responsibilities.
- **Knowledge half-life**: documentation decays; systems need refresh triggers and ownership—not guilt-driven updates.
- **Feedback loops shape behavior**: fast tests and clear signals outperform policy memos.
- **Psychological safety enables accuracy**: blameless environments surface defects earlier; architectural learning compounds.

## Learning Path & Maturity Stages

1. **Clarity foundations** — Naming, module boundaries, and discoverable architecture diagrams tied to reality.
2. **Learning rituals** — Tech talks, reading clubs, and incident reviews oriented toward patterns—not personalities.
3. **Tooling ergonomics** — Golden paths that reduce decision fatigue without eliminating escape valves.
4. **Maturity instrumentation** — Surveys and telemetry on friction points, correlated with delivery outcomes.
