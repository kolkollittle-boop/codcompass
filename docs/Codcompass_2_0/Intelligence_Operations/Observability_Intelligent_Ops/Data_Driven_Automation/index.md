# Data-Driven Automation

## Core Theory & Mental Models

- **Decision latency**: automation value equals human time saved minus risk of wrong actions—net present outcome matters.
- **Data contracts upstream**: automation fails silently when feeds lack semantics, lineage, and SLA guarantees.
- **Guardrail taxonomy**: rate limits, dual control, simulation environments, and circuit breakers compose defense in depth.
- **Explainability bar**: stakeholders must understand *why* automation acted—or trust erodes faster than efficiency gains.

## Learning Path & Maturity Stages

1. **Instrument before automate** — Measure variance, seasonality, and edge cases in proposed decisions.
2. **Shadow mode** — Parallel decisions without side effects; compare to human baselines.
3. **Graduated autonomy** — Narrow scopes, reversible actions, expand only with proven calibration.
4. **Lifecycle governance** — Model/pipeline retirement, drift monitors, and periodic human audits.
