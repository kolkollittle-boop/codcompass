# Agentic Systems

## Core Theory & Mental Models

- **Agency spectrum**: deterministic automation vs. goal-directed behavior with bounded autonomy; explicit delegation boundaries.
- **Control planes vs. data planes**: where policy, evaluation, and rollback live versus where execution occurs.
- **Failure taxonomy**: tool errors, reasoning drift, coordination deadlock, and ambiguous objectives—each demands different containment.
- **Composition over monolith intelligence**: small, testable capabilities composed through contracts, not opaque prompts.

## Learning Path & Maturity Stages

1. **Foundations** — Define objectives, constraints, and success signals before expanding tool surfaces.
2. **Structured autonomy** — Introduce policies, staged rollouts, and human-in-the-loop gates for irreversible actions.
3. **Orchestration maturity** — Multi-actor workflows with explicit handoffs, shared state boundaries, and regression suites for behavior.
4. **Industrialization** — Evaluation harnesses, incident playbooks, and continuous alignment with system evolution (see sibling topics).
