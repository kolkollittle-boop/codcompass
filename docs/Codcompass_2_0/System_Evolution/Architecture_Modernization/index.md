# Architecture Modernization

## Core Theory & Mental Models

- **Strangler patterns**: incremental replacement beats big-bang when behavioral equivalence is uncertain.
- **Semantic coupling**: the real cost of legacy is often implicit knowledge, not syntax—modernization plans must surface invariants.
- **Architectural fitness functions**: measurable properties (deployability, modularity, security posture) steer evolution objectively.
- **Migration as product**: roadmap, risk budgets, and stakeholder alignment are first-class—not afterthoughts to coding.

## Learning Path & Maturity Stages

1. **Discovery & invariants** — Map domains, boundaries, data gravity, and non-negotiable constraints.
2. **Pilot seams** — Introduce stable interfaces that allow dual-running old and new paths safely.
3. **Scaled migration** — Repeatable playbooks, automated verification, and rollback-ready milestones.
4. **Steady-state governance** — Prevent re-legacy through reviews, fitness functions, and ownership clarity.
