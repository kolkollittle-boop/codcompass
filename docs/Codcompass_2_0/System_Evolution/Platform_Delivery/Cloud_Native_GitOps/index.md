# Cloud Native & GitOps

## Core Theory & Mental Models

- **Declarative desired state**: reconciliation loops reward explicit drift detection over imperative snowflakes.
- **Git as contract hub**: policies, manifests, and audits converge when change management is traceable.
- **Progressive delivery**: traffic management and feature flags separate release from exposure—critical for cloud-native blast radius.
- **Shared responsibility clarity**: platform vs. workload ownership for networking, identity, and secrets must be documented.

## Learning Path & Maturity Stages

1. **GitOps hygiene** — Repo structure, promotion flows, and secrets handling without leaking plaintext patterns.
2. **Policy-as-code** — Admission controls, supply-chain attestations, and automated exceptions workflows.
3. **Operational maturity** — Runbooks keyed to controllers, observability of reconcile loops, and disaster recovery drills.
4. **Multi-environment governance** — Parity standards, test data boundaries, and promotion criteria under regulatory constraints.
