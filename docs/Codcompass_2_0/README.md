# Codcompass 2.0 — Knowledge Scaffold

## Philosophy

Codcompass 2.0 is a **decision-oriented** knowledge architecture for senior engineers and technical leaders. It is organized around one recurring question:

**“In this situation, how should I decide—and how should the system evolve?”**

Content favors **patterns**, **trade-offs**, **decision frameworks**, and **evolution paths** over tool tutorials or one-off stories. Categories are designed for **high cohesion within pillars** and **low coupling across pillars**, so you can navigate by problem domain without forcing artificial boundaries.

## Navigation Logic

| Pillar | Architect-level intent |
|--------|-------------------------|
| **AI_Engineering** | How do we design, deploy, and compound intelligence safely and sustainably? |
| **System_Evolution** | How do platforms modernize, scale, and deliver without ossifying? |
| **Intelligence_Operations** | How do discovery, visibility, and operational feedback loops improve outcomes? |
| **Architectural_Mastery** | How do we reason explicitly, learn systematically, and raise organizational maturity? |
| **Cross_Cutting_Concerns** | What constraints cut across domains—security, economics, platform telemetry—and how do we balance them? |

**Archive** holds superseded or frozen snapshots of thinking. **References** points to external canon (principles, papers, timeless articles)—not vendor collateral.

## How to Use This Tree

1. Enter the pillar that matches your **decision context** (not your job title).
2. Read the parent **`index.md`** for mental models and maturity framing.
3. Drill into leaf topics for **pattern vocabulary** and **staging** guidance.
4. Prefer **ADRs** and explicit trade-offs when committing direction.

## Naming Conventions

- Folders use **PascalCase** or **Upper_Snake_Case** for portability (Obsidian, Cursor, Git).
- No numeric prefixes; ordering is conceptual, not lexical.
- No locale-specific folder names; prose inside files is English for this scaffold.
