# Observability Platforms

## Core Theory & Mental Models

- **Platform complements operations**: this pillar defines **shared telemetry contracts, retention, privacy boundaries, and vendor-neutral schemas**—while **Intelligence_Operations / Observability_Intelligent_Ops** focuses on **closing feedback loops and intelligent remediation patterns**.
- **Cardinality discipline**: unconstrained labels explode cost and obscure signal—platforms enforce budgets.
- **Portable semantics**: open telemetry standards and consistent naming enable migration and multi-cloud sobriety.
- **Privacy-by-design pipelines**: sampling, redaction, and access scopes are architectural—not log-shipper afterthoughts.

## Learning Path & Maturity Stages

1. **Standards adoption** — Unified instrumentation libraries and mandatory context propagation.
2. **Platform services** — Managed collectors, query interfaces, and SLO tooling as internal products.
3. **Governance** — Data retention classes, PII handling, and cross-team usage policies.
4. **Ecosystem evolution** — Plugin models for new signals without fragmenting the canonical observability graph.
