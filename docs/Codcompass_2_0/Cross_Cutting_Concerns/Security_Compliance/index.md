# Security & Compliance

## Core Theory & Mental Models

- **Defense in depth**: no single control suffices; layers fail independently—design assumes partial compromise.
- **Least privilege dynamics**: static roles drift; continuous entitlement review is architectural, not audit theater.
- **Compliance as constraints**: regulations encode societal risk tolerance—translate into testable controls and data boundaries.
- **Secure defaults**: platforms beat policies when dangerous paths require explicit opt-in.

## Learning Path & Maturity Stages

1. **Threat modeling rhythm** — Repeatable STRIDE-style analysis tied to architecture deltas.
2. **Identity & secrets posture** — Rotation, blast-radius containment, and breach rehearsal—not checkbox IAM matrices.
3. **Secure SDLC integration** — Supply-chain attestations, dependency strategy, and vulnerability SLAs.
4. **Assurance maturity** — Evidence pipelines for auditors and engineers alike; continuous control monitoring.
