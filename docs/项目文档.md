## 项目概述

### 愿景

打造一个面向海外开发者的**高质量技术知识库付费平台**，通过 AI 驱动的多源整合与原创内容，提供独特视角的技术教程和实战指南。

### 核心价值主张（v1.1 修订）

| 价值点 | 描述 |
|--------|------|
| 🎯 **精准定位** | 聚焦英语/中文开发者市场，定价 $9.99-29/月（先低价验证再涨价） |
| 🤖 **AI 驱动** | 基于 OpenClaw + Claude Code 自动化内容生产 |
| 📚 **多源整合 + 原创** | 每篇文章整合 3-5 个来源，加入原创实战案例和个人见解 |
| 🔒 **合规优先** | 严格遵循合理使用原则，多源融合+结构重组+引用标注 |
| 💎 **简约专业** | 参考 Stripe/Impeccable 设计风格，内容为主 |

### 商业模式（v1.1 修订 — 降价验证）

```
免费层 → 基础会员 ($9.99/月) → 专业会员 ($29/月) → 企业会员 ($49/月)
   ↓           ↓                    ↓                    ↓
 预览内容   完整教程             + 源码下载          + 1v1 咨询
 部分文章   + 代码示例           + 视频教程          + 定制开发
```

> 💡 **定价策略**: MVP 阶段 $9.99/月验证 PMF，100+ 付费用户后逐步涨至 $29/月

## 目标市场分析

### 目标用户画像

| 维度 | 描述 |
|------|------|
| **地理位置** | 美国、加拿大、英国、德国、阿联酋（迪拜） |
| **职业** | 软件工程师、技术负责人、独立开发者、CTO |

## 爬虫系统设计（MVP 简化版）

> ⚠️ **MVP 阶段不需要 BullMQ + Redis + 独立 Worker**。
> MVP 只需：Vercel Cron 定时触发 → Next.js API Route 处理 → PostgreSQL 存储。
> 详细设计见 [05-爬虫系统设计.md](./05-爬虫系统设计.md)。

### MVP 数据源（优先用 API，不碰 HTML 爬取）

| 优先级 | 数据源 | 方式 | 频率 | 反爬风险 |
|--------|--------|------|------|----------|
| 🔴 P0 | GitHub Trending | HTML + cheerio | 每 2h | 低 |
| 🔴 P0 | Hacker News | Firebase API | 每 2h | 无 |
| 🔴 P0 | Reddit r/programming | Reddit JSON API | 每 2h | 低 |
| 🟡 P1 | Dev.to | 官方 REST API | 每 6h | 无 |
| 🟡 P1 | Hashnode | GraphQL API | 每 6h | 低 |
| 🟡 P1 | Lobsters | JSON 端点 | 每 6h | 无 |
| 🟢 P2 | 官方文档 | HTML 解析 | 每天 | 低 |

**MVP 阶段不碰：** YouTube（需 API key）、Twitter/X（API 收费）、Medium 付费墙

### MVP 反爬策略

| 策略 | 实现 |
|------|------|
| **本机代理** | 本机 Clash/Surge (127.0.0.1:7890)，环境变量控制 |
| **User-Agent** | 统一标识 `CyberPunkKB/1.0` |
| **请求间隔** | 2-3 秒随机延迟 |
| **API 优先** | 优先用官方 API，HTML 解析仅备用 |

## 内容处理流程（v1.1 修订）

### 内容生产 Pipeline（多源整合版）

```
多源抓取 → 提取 → 去重 → 多源整合 → AI 重写 → 原创注入 → 人工审核 → 发布
   ↓         ↓       ↓        ↓           ↓         ↓           ↓           ↓
 API/HTML  清洗    SimHash  3-5源合并   Claude    实战案例    编辑把关    网站
         标准化   检测     结构重组    改写      个人见解   质量评分   + SEO
```

### 避免抄袭策略（核心：多源整合 + 原创）

| 步骤 | 方法 | 工具 |
|------|------|------|
| **1. 多源融合** | 同一主题整合 3-5 个不同来源 | 自定义脚本 |
| **2. 结构重组** | 完全重新组织结构，不按原文 | AI 重写 |
| **3. 原创注入** | 添加个人见解、实战案例、代码示例 | Claude Code |
| **4. 代码重写** | 所有示例代码重新实现 | Claude Code |
| **5. 相似度检测** | 发布前查重 | Copyscape API |
| **6. 引用规范** | 每篇文章末尾标注所有参考来源 | 自动引用系统 |
| **7. 人工审核** | MVP 阶段前 20 篇必须人工审核 | 编辑把关 |

### 内容风格指南（简约专业版）

```markdown
# 技术写作风格（参考 Stripe 技术博客风格）

## 语气
- 直接、专业、不废话
- 像高级工程师在跟你对话
- 用"你"而不是"用户"，建立直接感

## 格式
- 短段落，每段不超过 4-5 行
- 大量可运行的代码示例
- 关键概念用 `code` 或 **bold** 强调
- 每个章节以 TL;DR 总结
- 每篇文章包含实战案例（Project-Based）

## 示例
> "今天我们来深入理解 XX 的工作原理。
> 不需要前置知识，准备好终端，我们直接开始。"
```

## 合规与法律

### 版权合规清单（v1.1 修订）

- [x] **多源整合原则**: 每篇文章必须整合 3-5 个不同来源，不做单篇改写
- [x] **结构重组**: 完全重新组织结构，不按原文顺序
- [x] **原创注入**: 每篇文章必须包含原创实战案例或个人见解
- [x] **来源标注**: 每篇文章末尾列出所有参考来源
- [x] **代码重写**: 所有示例代码必须重新实现，不直接复制
- [x] **合理使用原则**: 引用不超过原文 10%，且添加实质性新内容
- [x] **图片授权**: 使用 CC0/CC-BY 图片，或自行拍摄/生成
- [x] **免责声明**: 明确说明教程仅供参考，不构成专业建议
- [x] **隐私政策**: GDPR/CCPA 合规的用户数据处理说明
- [x] **服务条款**: 明确会员权益、退款政策

### 法律实体建议

```
推荐注册地：美国特拉华州 LLC 或 英国 Ltd
原因:
- 对海外业务友好
- 税收优化
- Lemon Squeezy / Stripe 支持良好
- 有限责任保护

备选：阿联酋迪拜 DMCC 自由区
- 0% 企业所得税
- 100% 外资所有权
- 适合服务中东市场
```

## 风险与应对（v1.1 修订）

### 技术风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 爬虫被封 | 低 | 中 | API 优先、本机代理、降频 |

| 支付集成问题 | 低 | 高 | 先用 Lemon Squeezy 测试模式 |
| 数据库性能瓶颈 | 低 | 低 | MVP 阶段 Supabase 免费额度足够 |

### 法律风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 版权投诉 | 中 | 高 | 多源整合+结构重组+引用标注，DMCA 响应流程 |
| GDPR 违规 | 低 | 高 | 隐私合规审计、数据最小化 |
| 商标侵权 | 低 | 中 | 品牌名称检索、避免使用知名商标 |

### 商业风险

| 风险 | 概率 | 影响 | 应对措施 |
